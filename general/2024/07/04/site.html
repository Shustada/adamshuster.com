Site Set-Up

This site was built using the code from another that I liked for its simplicity and functionality. I've made small code edits here and there but kept the structure mostly the same. The website is hosted on a Bluehost server and is set up with a remote git repository.

I enjoy using git to update the page because it simplifies the process. All I have to do is make changes in my desktop directory—like adding a post or making a change—and then use the terminal to push the changes to both my GitHub and Bluehost’s server. The website updates, and I can see the changes at my URL. All changes are cataloged and copied in my GitHub repository.

The issue is that since everything is in HTML, anytime I write a post and want to upload it, I have to convert the markdown to HTML using a converter, create a new HTML page by copying older blog formats, add the converted markdown, and then update the git repository. This whole process takes around 10-15 minutes. Ideally, I want to simplify this using a tool like Jekyll or something similar.

Site's Purpose

I need to think about what I want this site to be and its purpose. Right now, it’s mostly just for myself. I enjoy seeing the finished product of what I create uploaded to a server. I don’t believe people want to read what I write, so I haven’t geared the site toward readers. However, I think I may eventually orient my site to be more professional or formal to appeal to audiences.
